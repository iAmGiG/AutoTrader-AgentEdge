# Future Reference Library

This folder contains ideas, papers, and tools that are worth tracking but don't warrant immediate GitHub issues. Items here are for future consideration when the project reaches the appropriate phase.

## Organization

| File | Purpose |
| ---- | ------- |
| [research_potential.md](research_potential.md) | Academic papers, strategies, ML techniques |
| [workflow_potential.md](workflow_potential.md) | Tools, processes, developer experience |

## When to Promote to GitHub Issue

Move an item to a GitHub issue when:

1. The project phase aligns with the idea
2. There's clear implementation scope
3. Dependencies are resolved
4. It becomes a priority

## Entry Template

Each entry should include:

- **Source**: Link or citation
- **Summary**: 2-3 sentence overview
- **Relevance**: How it applies to AutoTrader
- **Priority**: Low / Medium / Future
- **Revisit When**: Trigger condition (e.g., "When building RiskAgent")
